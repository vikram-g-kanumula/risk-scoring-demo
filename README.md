# US Homeowners Risk Scoring Engine

### Business Development Capability Demo

**A GLM-first risk scoring architecture that captures compounding peril risk through an ML interaction layer trained on GLM residuals — with exact regulatory transparency at every layer.**

---

## Quick Start

```bash
# 1. Unzip and enter the project directory
unzip risk-scoring-demo.zip
cd risk-scoring-demo

# 2. Create a virtual environment (recommended)
python -m venv venv
source venv/bin/activate        # macOS / Linux
# venv\Scripts\activate         # Windows

# 3. Install dependencies (~2 min)
pip install -r requirements.txt

# 4. Generate data and train models (~6–11 min, one-time only)
python setup.py

# 5. Launch the demo
streamlit run app.py
```

The app opens at `http://localhost:8501`.

## Architecture

The core claim of this demo — that compound-peril interactions are systematically under-priced by additive models — is supported by a specific, auditable pipeline. The GLM is the causal anchor; M̂ is defined entirely in terms of the GLM's residuals.

### The Four-Step GLM Improver Pattern

This architecture implements the Dai (CAS E-Forum, Spring 2018) GLM Improver pattern:

```
Step 1 — GLM Baseline (Regulatory Layer)
  PoissonRegressor(log-link)  → λ_GLM  (annual claim frequency)
  GammaRegressor(log-link)    → μ_GLM  (expected severity | claim occurred)
  Baseline E[L] = λ_GLM × μ_GLM
  Features: 19 Tier 1 + Tier 2 variables
  Output: DOI-auditable exp(β) rate relativities

Step 2 — GLM Residual (M̂ Training Target)
  M_actual(i) = claim_amount(i) / max(GLM_prediction(i), floor)
  M_actual > 1.0  →  GLM under-predicted (compound risk present)
  M_actual = 1.0  →  GLM was accurate
  M_actual < 1.0  →  GLM over-predicted (protective interactions)

Step 3 — M̂ Ensemble (Interaction Discovery Layer)
  Input: Tier 3 features only
         (wildfire_zone, roof_material, flood_zone, dist_to_coast_mi,
          construction_type, roof_age_yr, earthquake_zone, state,
          slope_steepness_pct, post_burn_rainfall_intensity, ...)
  Models: RandomForest + HistGradientBoosting + ExtraTrees
  Meta:   Ridge + IsotonicRegression (OOF stacked)
  Target: M_actual — what the GLM left unexplained

Step 4 — Final Pricing
  E[L](i) = λ_GLM(i)  ×  μ_GLM(i)  ×  M̂(i)
  When M̂ = 1.0  →  reduces exactly to the pure GLM baseline
  Premium = E[L] / target_loss_ratio × expense_factor
```

### Display Artifact

A `TweedieRegressor(p=1.65)` is also trained and stored in `artifacts.pkl`. It is **not** used in the live pricing chain. It exists as the DOI rate-filing display artifact — its exp(β) coefficients are shown in the Methodology tab's regulatory comparison table.

## The M̂ Table — Named Interaction Effects

These are the compound-peril interactions the M̂ ensemble discovers from GLM residuals. Each has a clear physical mechanism:

| Interaction | Representative M̂ | Actuarial Logic |
|-------------|------------------|----------------|
| Wood Shake × High Wildfire Zone | ×3.0–3.5 | Embers ignite degraded shingles at 572°F vs 700°F for asphalt. Camp Fire: 85% of burned structures had WUI + aged roof co-exposure |
| High Flood Zone × Coastal <5mi | ×2.0–2.5 | Storm surge amplification. Non-linear depth × proximity interaction in hurricane-track states |
| Wood Shake × Moderate Wildfire | ×1.8–2.1 | Partial ember transport distance; significantly elevated vs baseline |
| Non-Wood × High Wildfire | ×1.5–1.8 | Even resilient roofs exposed to extreme fire behaviour at WUI; ember intrusion through vents and eaves |
| High Earthquake Zone | ×1.3–1.6 | Seismic + structural age compound. Pre-1970 construction most sensitive |
| Old Roof (>20yr) × Frame Construction | ×1.2–1.4 | UV granule loss + combustible frame = water intrusion + fire severity cycle |
| Water Claims ×2+ × High Vegetation | ×1.2–1.3 | Root systems seek moisture, penetrating foundations and sewer lines; prior water claims confirm the cycle |
| Old Roof (>20yr) × High Hail Zone | ×1.2–1.3 | Cumulative micro-damage from repeated hail accelerates granule loss; functional failure at lower hail sizes |
| Frame × Protection Class ≥8 | ×1.1–1.2 | Slow fire response + fast-igniting frame = severity amplification |
| Slope >55% × High WF × Post-Burn Rain >60 | ×1.3–1.5 | Montecito 2018 archetype: Thomas Fire debris flow — 23 deaths, 400+ homes |
| Defensible Space ≥80 + Metal Roof | ×0.80–0.90 | Protective synergy — learned from data, not hard-coded. CA Reg 2644.9 mitigation credit |

*Specific values depend on training run. Ranges reflect expected M̂ from residual-trained ensemble on synthetic data.*

## Troubleshooting

**`KeyError` on app startup (e.g. `KeyError: 'poisson_glm'`)**
The `artifacts.pkl` is missing or was built by an incompatible trainer. Re-run `python setup.py` to regenerate.

**`FileNotFoundError: models/artifacts.pkl`**
`setup.py` has not been run or failed partway. Check terminal output and re-run after fixing.

**`ModuleNotFoundError` on any import**
Run `pip install -r requirements.txt` before `python setup.py`.

**LightGBM install fails on Apple Silicon**
```bash
pip install lightgbm --no-binary lightgbm
# or
conda install lightgbm
```

**SHAP values show as unavailable in Tab 5**
Ensure `shap>=0.44.0` is installed. The GLM LinearExplainer and M̂ TreeExplainer are loaded lazily — if shap is missing, Tab 5 falls back to model-native feature importances without crashing the rest of the app.

**Choropleth map appears blank**
Plotly's `go.Choropleth` with `locationmode="USA-states"` requires an active Plotly.js renderer. If running in a headless or restricted environment, ensure Streamlit's Plotly integration is functioning. The chart renders correctly in all standard browser environments.

**Portfolio analytics slow on first load**
The app pre-computes M̂ distributions, batch predictions, and reclassification analytics on first load (~10 seconds). These are cached in `st.session_state` and persist for the browser session.
